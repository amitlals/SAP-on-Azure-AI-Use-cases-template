# Contributing to SAP on Azure AI Use Cases Template

Thank you for your interest in contributing to this template repository! We welcome contributions from the community.

## üåü How Can You Contribute?

### 1. Report Issues
- Found a bug or have a suggestion? [Open an issue](https://github.com/amitlals/SAP-on-Azure-AI-Use-cases-template/issues)
- Please provide as much detail as possible
- Include screenshots or examples when applicable

### 2. Submit Use Cases
- Share your SAP + Azure AI integration scenarios
- Include clear documentation and code examples
- Follow the existing template structure

### 3. Improve Documentation
- Fix typos or clarify existing documentation
- Add missing details or examples
- Translate documentation to other languages

### 4. Code Contributions
- Fork the repository
- Create a feature branch (`git checkout -b feature/YourFeature`)
- Make your changes
- Test your changes thoroughly
- Commit with clear messages (`git commit -m 'Add: New use case for Azure OpenAI'`)
- Push to your branch (`git push origin feature/YourFeature`)
- Open a Pull Request

## üìã Contribution Guidelines

### Code Quality
- Follow existing code style and conventions
- Include comments for complex logic
- Ensure code is well-documented
- Test your changes before submitting

### Documentation
- Use clear, concise language
- Include code examples where applicable
- Add screenshots or diagrams for visual context
- Keep documentation up-to-date with code changes

### Commit Messages
- Use clear, descriptive commit messages
- Start with a verb (Add, Fix, Update, Remove)
- Keep the first line under 50 characters
- Add detailed description if needed

### Pull Request Process
1. Update the README.md with details of changes if applicable
2. Ensure your code follows the project's coding standards
3. Your PR will be reviewed by maintainers
4. Address any feedback or requested changes
5. Once approved, your PR will be merged
